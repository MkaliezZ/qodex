# Milestone M9 — MCP Runtime Foundation

## Purpose

Introduce MCP (Model Context Protocol) support as the standard mechanism for external capabilities. MCP tools provide filesystem, Git, and terminal operations through a permission-gated runtime.

## Architecture Added

```
MCPRuntime.initialize()
    ↓
3 built-in MCP Servers registered
    ├── mock-filesystem (3 tools: read_file, write_file, list_directory)
    ├── mock-git        (3 tools: git_status, git_diff, git_log)
    └── mock-terminal   (1 tool: run_command)
    ↓
Tool Discovery → Registry → Permission Gate → Transport → Call Result
    │                                │
 MCPEventBus                   PermissionEngine
 server.connected/disconnected   ask | allow_once
 tool.discovered/called          | allow_session | deny
 permission.requested
```

## Permission Engine

| Mode | Behavior |
|:--|:--|
| `ask` | **Default** — Prompt user for confirmation each time |
| `allow_once` | Grant for single call |
| `allow_session` | Grant for entire session (cached) |
| `deny` | Reject all tool calls |

Every tool call must pass through the permission engine. No tool executes automatically.

## Runtime Events

| Event | When |
|:--|:--|
| `mcp.server.connected` | Server registered |
| `mcp.server.disconnected` | Server removed |
| `mcp.tool.discovered` | Tool registered |
| `mcp.tool.called` | Tool executed |
| `mcp.permission.requested` | Permission checked |

## Major Decisions

1. **Permission-first architecture**: Every tool call passes through `PermissionEngine.request()` before reaching the transport layer. The default mode is `ask` — the safest default.

2. **Transport adapter pattern**: `MockTransport` for development/testing. `StdioTransport` ready for Tauri production (actual `git` subprocess communication).

3. **Mock servers only**: No real filesystem operations, no real Git commands, no real shell execution. All responses are mock data. Real execution will require the Tauri adapter layer.

4. **No remote transport**: M9 explicitly omits WebSocket, HTTP, and any remote MCP transport. Local stdio only.

## Production Review

**Result: PASSED** — All 18 scenarios validated. 692 total tests (all 8 packages). Zero defects found.

## Known Limitations

- Mock transport only — no real MCP server communication
- No stdio subprocess implementation yet
- No remote MCP server support
- No tool parameter validation beyond what MockTransport provides
- No tool execution rate limiting
