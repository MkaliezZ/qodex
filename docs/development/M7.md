# Milestone M7 — Git Runtime Foundation

## Purpose

Introduce project-level safety and recovery through Git-based checkpoints, commits, and branches. Every applied change becomes recoverable without requiring GitHub or remote operations.

## Architecture Added

```
GitRuntime
    ├── GitAdapter (abstract)
    │    ├── MockGitAdapter (in-memory for testing)
    │    └── SimpleGitAdapter (future: real git CLI)
    │
    ├── CheckpointEngine: create/restore/list/remove/clear
    ├── CommitEngine: create/list/getLatest
    ├── BranchEngine: create/switch/list/getCurrent
    ├── StatusEngine: getStatus/hasChanges/getFileCount
    └── GitEventBus: checkpoint.created/restored, commit.created, branch.created/switched
```

## Core Interfaces

| Interface | Purpose |
|:--|:--|
| `GitRepository` | Repository metadata: rootPath, isInitialized, currentBranch |
| `GitCheckpoint` | Named restore point: id, name, commitHash, createdAt |
| `GitCommit` | Commit record: hash, message, createdAt |
| `GitStatus` | File state: modified, added, deleted, untracked |
| `GitBranch` | Branch info: name, isCurrent, commitCount |
| `GitEvent` | Event for UI: checkpoint/commit/branch events |

## Checkpoint Lifecycle

```
Apply Patch → createCheckpoint("pre-release") → Modify more → restoreCheckpoint("pre-release") → Files restored
     │                                            │                    │
  checkpoint.created                         checkpoint.restored
```

## Major Decisions

1. **Adapter pattern for Git**: The `GitAdapter` interface abstracts all `git` operations. The `MockGitAdapter` is the primary implementation (in-memory). A `SimpleGitAdapter` that shells out to `git` CLI will be added for the Tauri environment.

2. **Checkpoints ≠ Commits**: Checkpoints are a user-friendly abstraction over Git commits. Users can create/revert checkpoints without understanding Git. Internally, each checkpoint maps to a commit hash.

3. **No remote operations**: By design, M7 explicitly prohibits `push`, `pull`, `fetch`, `remote`. Local-only Git foundation.

4. **No auto-commits**: Checkpoints and commits require explicit user action. The Diff Engine's apply does not auto-create checkpoints.

## Known Limitations

- Mock adapter only — no real `git` CLI integration yet
- No rebase, merge, or cherry-pick support
- No Git diff integration with Diff Engine
- No remote repository support
