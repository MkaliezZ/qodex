# Milestone M2 — Provider SDK Foundation

## Purpose

Build the provider abstraction layer that all future model integrations use. Every model — OpenAI, DeepSeek, Claude, Gemini, and all Chinese vendors — must go through this SDK.

## Architecture Added

```
Provider Registry → Base Provider → Provider Implementations
       │                                  ├── OpenAIProvider
       │                                  ├── DeepSeekProvider
       │                                  ├── OpenRouterProvider
       │                                  └── CustomProvider
       │
StreamManager ← SSE Parser ← HTTP Client
    │
Error Layer (6 canonical types)
```

## Core Interfaces

| Interface | Purpose |
|:--|:--|
| `ModelProvider` | Abstract provider interface (listModels, stream, testConnection) |
| `ModelInfo` | Model metadata (context window, tool/vision/reasoning support) |
| `ModelRequest` | Canonical request format (messages, tools, temperature) |
| `ModelChunk` | Unified stream chunk (text | tool_call | usage | error) |
| `ProviderRegistry` | Registry for provider registration, lookup, lifecycle |
| `StreamManager` | SSE stream normalization where all providers emit identical chunks |
| `ProviderError` | 6 canonical error types (auth_error, rate_limit, network_error, model_not_found, invalid_request, unknown) |

## Major Decisions

1. **All providers share `BaseOpenAICompatibleProvider`**: Both DeepSeek and OpenRouter use OpenAI's `/chat/completions` wire format. This eliminates redundant implementations.

2. **`openai-chat` protocol as the baseline**: Even future providers like Qwen, GLM, and MiniMax support this protocol. Only Anthropic and Gemini will need dedicated adapters.

3. **No external dependencies**: Pure TypeScript using the built-in `fetch` API (Node 18+). No axios, no graphql-client.

4. **API keys never logged**: The `authHeaders()` method builds headers inline; keys are stored in memory and never included in error output.

## Known Limitations

- All provider tests use mock network responses
- No real API calls are made during testing
- No streaming UI integration yet
