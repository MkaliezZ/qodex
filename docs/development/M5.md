# Milestone M5 — Context Engine Foundation

## Purpose

Replace naive prompt+file concatenation with a formal Context Engine that assembles project rules, session memory, project metadata, selected files, and the user's prompt into a structured ContextBundle.

## Architecture Added

```
ContextEngine.buildContext(ContextRequest)
    ↓
RulesLoader.load()     ← qodex-config/rules.md
MemoryLoader.load()    ← qodex-config/memory.md
ProjectMetadata        ← ProjectIndex summary
FileContextBuilder     ← selected FileContent[]
Original Prompt
    ↓
TokenEstimator.estimate()
    ↓
ContextBundle → AgentRuntime.runTask()
```

## Core Interfaces

| Interface | Purpose |
|:--|:--|
| `ContextRequest` | Input: prompt, selectedFiles, projectId, maxTokens |
| `ContextBundle` | Output: assembledPrompt, sources, estimatedTokens |
| `ContextSources` | Individual sections: prompt, rules, memory, metadata, files |

## Assembly Order

```
=== Project Rules ===        ← RulesLoader (qodex-config/rules.md)
=== Session Memory ===       ← MemoryLoader (qodex-config/memory.md)
=== Skills ===               ← SkillRuntime (added in M8)
=== Project Metadata ===     ← ProjectMetadataBuilder
=== Selected Files ===       ← FileContextBuilder
=== Task ===                 ← Original user prompt (ALWAYS last)
```

## Token Estimation

Lightweight estimator: ~4 chars/token for ASCII, ~2 chars/token for CJK characters. No model-specific tokenizer — approximation only.

## Major Decisions

1. **Default in-memory rules/memory**: In browser dev mode, rules and memory are provided as embedded strings. Tauri will replace with file readers.

2. **Prompt always last**: The user's original task prompt is always the final section in the assembly. This ensures the model sees the task immediately after context.

3. **No chunking or ranking**: M5 is assembly-only. Chunking and ranking belong to later phases.

## Production Review

**Result: PASSED** — All scenarios validated (10 scenarios, token estimator validated across small/medium/large).

## Known Limitations

- No semantic search (embeddings planned for later)
- No context compression
- No Git diff context
- No import graph or symbol index
- Token estimation is approximate (±20%)
