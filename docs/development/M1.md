# Milestone M1 — Desktop Shell

## Purpose

Create the Tauri + React + TypeScript desktop application shell with a professional three-column layout and dark glassmorphism design.

## Architecture Added

- **Layout**: Three-column (Left Rail 220px | Center elastic | Right Panel 280px)
- **Design system**: Dark glassmorphism with fluid gradient background
- **Components**: 8 React components for the complete shell

## Packages Added

| Package | Path | Purpose |
|:--|:--|:--|
| `@qodex/desktop` | `apps/desktop/` | Tauri + React frontend |

### Components

| Component | Location | Purpose |
|:--|:--|:--|
| `AppShell` | Center | Root layout with Context Provider |
| `ProjectRail` | Left panel | Navigation + project tree |
| `AgentTimeline` | Center top | Agent response display |
| `PromptBar` | Center bottom | Input + controls |
| `ModelSwitcher` | PromptBar inline | Model selection |
| `SkillDrawer` | PromptBar inline | Skill insertion |
| `ContextPanel` | Right panel | Config + status |
| `DiffViewer` | Timeline bottom | Patch review |

## Tests Added

None (UI shell only — no logic to test).

## Major Decisions

1. **Tauri 2 over Electron**: Rust-based Tauri provides smaller binary size, better performance, and native file system access.

2. **Vite dev server on port 1420**: Standard Tauri convention with HMR support.

3. **No CSS framework**: Pure CSS design system (~300 lines) rather than Tailwind/MUI. Full control over glassmorphism effects.

4. **240px → 220px left rail**: Width reduced during M1.1 polish for better balance.

5. **320px → 280px right panel**: Reduced during M1.1 to eliminate wasted whitespace.

## UI Polish (M1.1)

After initial M1 completion, a polish pass improved visual quality:

- Removed all emoji icons from navigation bar
- Replaced emoji with monochrome text/symbols
- Reduced ContextPanel card borders and colored backgrounds
- Improved typography hierarchy with section labels in sentence case
- Increased section spacing from 14px to 30px
- Added CSS spinner indicator for Run button loading state
- Refined Diff Preview empty state (removed 📝 icon, raised text position)
- All transitions unified to 120ms/160ms timings

## Known Limitations

- No actual Tauri compilation — Vite dev server only
- No real Model/Provider integration — pure UI shell
- Mock data for file tree and context panel
