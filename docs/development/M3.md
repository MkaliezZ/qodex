# Milestone M3 — Agent Runtime Skeleton

## Purpose

Create the first working end-to-end execution flow: PromptBar → AgentRuntime → Provider → Streaming Output → AgentTimeline.

## Architecture Added

```
PromptBar → AgentRuntime.runTask()
    ↓
TaskStateMachine (Idle → Planning → CallingModel → Streaming → Done)
    ↓
EventBus (publish/subscribe)
    ↓
AgentTimeline (live stream rendering)
```

## Packages Added

| Package | Path | Tests |
|:--|:--:|:--:|
| `@qodex/agent-runtime` | `packages/agent-runtime/` | 50 |

## Core Components

| Component | Purpose |
|:--|:--|
| `AgentRuntime` | Core orchestration — createSession, createTask, runTask, cancelTask |
| `TaskStateMachine` | 7-state machine with validated transitions |
| `EventBus` | Lightweight pub/sub for runtime-to-UI communication |
| `InMemorySessionStore` | Ephemeral session storage |
| `InMemoryTaskStore` | Ephemeral task storage with output appending |
| `MockStreamingProvider` | Generates mock text chunks without API keys |

## State Machine

```
Idle → Planning → CallingModel → Streaming → Done
                                        ↓
                                    Cancelled / Failed
```

## Events

| Event | Payload |
|:--|:--|
| `task.started` | { task } |
| `task.status_changed` | { status, previousStatus } |
| `message.chunk` | { text } |
| `task.completed` | { task } |
| `task.failed` | { error } |
| `task.cancelled` | { task } |

## Desktop Integration

- **PromptBar**: Run button wired to `runtime.createTask()` + `runtime.runTask()`, Enter key support
- **AgentTimeline**: Subscribes to `message.chunk` events, renders text live with blinking cursor
- **AppShell**: `RuntimeContext.Provider` passes runtime state to all child components

## Tests Added

- Event Bus: 8 tests (subscribe, unsubscribe, clear, exception safety)
- State Machine: 10 tests (all 6 state transitions, terminal detection)
- Session Store: 6 tests (create, get, list, remove, clear)
- Task Store: 6 tests (create, status update, append output, filter)
- Runtime: 7 tests (full lifecycle, cancellation, missing task)
- Mock Provider: 5 tests (models, streaming, testConnection)

## Major Decisions

1. **Pure runtime, no UI code**: The agent-runtime package contains zero React or Tauri imports. All UI integration happens through the EventBus.

2. **In-memory storage only**: No database for M3. Sessions and tasks are ephemeral — lost on page refresh. This is intentional for the skeleton milestone.

3. **MockProvider as default**: The runtime auto-registers a MockStreamingProvider on construction. No API keys required for development.

## Known Limitations

- No persistence (in-memory only)
- No project context
- No model selector integration — always uses MockProvider
- Stream output is pre-defined mock text, not from a real model
