# Milestone M8 — Skill Runtime Foundation

## Purpose

Introduce reusable Skills that inject domain-specific context into every agent task. Skills are local markdown files with metadata — no code execution, no plugins, no marketplace.

## Architecture Added

```
SkillRuntime.initialize()
    ↓
SkillLoader.loadAllSkills()   → BuiltInDataProvider
    ↓
SkillRegistry.register()      → Searchable by id/name
    ↓
SkillResolver.resolve(prompt) → Keyword matching (no embeddings)
    ↓
SkillRuntime.buildSkillSection() → "=== Skills ===" block
    ↓
ContextEngine integration      → Between Memory and Metadata
```

Each skill consists of `skill.json` (metadata) and `SKILL.md` (guidelines/checklist).

## Skill Resolver

Simple keyword matching: the resolver checks if any of a skill's `name`, `description`, or `tags` appear in the user's prompt (case-insensitive).

- "Review this React component" → `react-review`
- "Refactor TypeScript service" → `typescript-refactor`
- "Hello world" → no skills matched

## Major Decisions

1. **Keyword matching, not embeddings**: The resolver uses simple string inclusion checks — no embeddings, no vector search, no LLM classification. This keeps skill resolution deterministic and fast.

2. **Built-in provider with file system ready**: The `BuiltInDataProvider` returns hardcoded skill data for development. A file-system provider can read from `qodex-config/skills/` in production.

3. **Skills inject context, not code**: Skills are markdown content that gets injected into the prompt. They do not execute code, call APIs, or modify behavior. This keeps the security model simple.

4. **Disabled by default**: `bug-hunter` ships disabled. Skills must be explicitly enabled to participate in resolution.

## Known Limitations

- No file-system skill loader (built-in only)
- No skill dependencies or composition
- No skill marketplace or remote registry
- No LLM-based skill selection
