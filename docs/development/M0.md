# Milestone M0 — Repository Bootstrap & Organization

## Purpose

Initialize the Qodex monorepo structure, organize 61 existing specification documents into a categorized documentation system, and establish the AI agent workspace.

## Architecture Added

- Monorepo root with `package.json` and `pnpm-workspace.yaml`
- `apps/` and `packages/` directory skeleton
- `docs/` structure with 9 categories (P0–P8)
- `qodex/` AI agent workspace (rules, memory, ADRs)
- Root configuration files (`.gitignore`, `.gitattributes`, `.editorconfig`)

## Packages Added

None. This milestone established the repository foundation only.

## Tests Added

None. M0 was a structural milestone.

## Major Decisions

1. **Flat docs structure**: Original P1–P8 directory hierarchy collapsed into a single `docs/` directory with subdirectories. This eliminated redundant nesting while preserving document categorization.

2. **`qodex/` over `.qodex/`**: The AI agent workspace directory was initially named `.qodex/` (hidden), then renamed to `qodex/` after user feedback. Hidden files add friction — the workspace should be visible.

3. **pnpm over npm/yarn**: pnpm workspace protocol (`workspace:*`) chosen for monorepo dependency management. Compatible with Tauri projects.

4. **Document index**: Generated a complete `DOCUMENT_INDEX.md` mapping all 61 markdown documents to their categories, with source provenance.
