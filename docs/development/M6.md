# Milestone M6 — Diff Engine Foundation

## Purpose

Enable safe code modifications through patch generation and review. The model never edits files directly — it only proposes changes through patches, and the user must explicitly Apply or Reject.

## Architecture Added

```
Agent Output → PatchProposal
    ↓
DiffGenerator.generateDiff() → DiffResult
    ├── PatchValidator.validate() → Conflict[]
    ├── ApplyEngine.apply()       → ApplyResult[]
    ├── ApplyEngine.reject()      → No file changes
    └── ApplyEngine.rollback()    → Restore original
```

## Core Interfaces

| Interface | Purpose |
|:--|:--|
| `PatchProposal` | Complete patch: id, taskId, summary, file patches |
| `PatchFile` | Single file: path, oldContent, newContent |
| `PatchHunk` | Unified diff hunk: oldStart, newStart, line changes |
| `DiffResult` | Per-file diff: additions, deletions, hunks |
| `ApplyResult` | Apply outcome: success, path, error, rollback data |
| `PatchConflict` | Conflict: type (file_not_found, content_mismatch, empty_patch) |

## Patch Lifecycle

```
createProposal() → validateProposal() → preview() → apply() / reject() → rollback()
     │                    │                │           │            │
     │                  Conflicts[0]      diff text   changes       restore
     │                    │                preview     applied       original
  PatchProposal         Conflict[]        string      ApplyResult[] ApplyResult[]
```

## Conflict Detection

| Type | Detected When |
|:--|:--|
| `file_not_found` | Target file does not exist on disk |
| `content_mismatch` | File content changed since patch creation |
| `line_mismatch` | (Future) hunk context lines don't match |
| `empty_patch` | oldContent === newContent |

## Major Decisions

1. **Pure TypeScript diff algorithm**: No external `diff` library. The LCS-based diff generator handles Unix/Windows newlines, special characters, and produces valid unified diff output.

2. **In-memory apply for browser**: In Vite dev mode, file writes go through an adapter that stores in a `Map`. Tauri will use real file system writes.

3. **Apply validation first**: `DiffEngine.apply()` runs validation before any write. If conflicts exist, apply is blocked with error details.
