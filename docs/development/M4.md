# Milestone M4 — Project Runtime

## Purpose

Allow Qodex to open and understand local projects. This milestone establishes project awareness: opening folders, building file trees, indexing files, reading contents, and passing selected files as context to the Agent Runtime.

## Architecture Added

```
FileSystemAdapter (abstract)
    ├── WebFileSystemAdapter  (browser File System Access API)
    └── MockFileSystemAdapter (in-memory for testing)

ProjectRuntime.openProject()
    ↓
TreeBuilder.buildTree()
    ↓
FileReader.readFiles() → FileContent[]
    ↓
ProjectIndexer.buildIndex()
    ↓
User selects files → readSelectedFilesAsContext()
```

## Core Components

| Component | Purpose |
|:--|:--|
| `ProjectRuntime` | Core orchestration — open, close, read, select |
| `TreeBuilder` | Recursive file tree builder with expand/collapse/select |
| `FileReader` | Text file reading + binary detection + language detection |
| `ProjectIndexer` | Lightweight file index (path, size, language, mtime) |
| `IgnoreRules` | Default ignore patterns (.git, node_modules, dist, *.lock, *.db) |

## Desktop Integration

- **ProjectRail**: "Open Project" button → `showDirectoryPicker()` → file tree rendered with click-to-select
- **ContextPanel**: Selected files count and total size display
- **PromptBar**: Selected files automatically prepended as context via `readFilesAsContext()`

## Ignore Rules

Default excludes: `.git`, `node_modules`, `dist`, `build`, `coverage`, `.next`, `.nuxt`, `.turbo`, `vendor`, `.cache`, `target`, hidden files (except `.gitignore`/`.editorconfig`), lock files, database files.

## Binary Detection

Returns `"Unsupported Binary File"` for: `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`, `.pdf`, `.ico`, `.ttf`, `.woff`, `.zip`, `.tar`, `.exe`, `.dll`, `.wasm`, and 20+ other binary extensions.

## Major Decisions

1. **Adapter pattern**: File system operations go through `FileSystemAdapter` interface. The browser implementation uses the File System Access API (`showDirectoryPicker`). A Tauri implementation can use its `fs` module later.

2. **Read-only by design**: M4 explicitly forbids file writing. Writing comes in M6 (Diff Engine).

3. **No automatic indexing**: The index is built on `openProject()`, not on a background watcher. File changes after opening are not tracked.

## Known Limitations

- Browser File System Access API requires secure context (localhost or HTTPS)
- No file watching — index is static after open
- No Git awareness for file status
- No large file handling (no chunking)
